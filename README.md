# Tallest Superhero

Скрипт для поиска самого высокого супергероя по заданному полу и наличию работы. Данные загружаются из публичного API [Superhero API](https://akabab.github.io/superhero-api/).

## Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/nem1sha/QA-python
cd <ИМЯ_ПАПКИ_ПРОЕКТА>
```

2. Создайте и активируйте виртуальное окружение (рекомендуется):

```bash
python -m venv venv
source venv/bin/activate  # Linux/MacOS
venv\Scripts\activate     # Windows
```
3. Установите зависимости:

```bash
pip install -r requirements.txt
```

## Запуск

Запуск основной программы:

```bash
python tallest_superhero.py
```

Запуск тестов:

```bash
pytest -v
```

## Принятые решения

* Наличие работы. В данных поле occupation может принимать значения "-" или пустую строку "". Оба этих значения интерпретируются как отсутствие работы.

* Нулевой рост. Запись "0 cm" в данных используется как техническая заглушка. Такие герои исключаются из отбора, чтобы не влиять на результат.

* Единицы измерения. В базе встречаются значения в сантиметрах (cm) и метрах (meters). Все значения приводятся к сантиметрам для корректного сравнения роста.

* Неизвестная единица. В данных присутствует некорректная запись "0 kg". Такие герои отсеиваются, чтобы избежать ошибок при сравнении.

* Равный максимальный рост. При одинаковом максимальном росте возвращается герой, встретившийся в данных первым. Это достигается за счет стабильности сортировки в Python.

* Возвращается имя. В API имена героев не являются уникальными идентификаторами (например, несколько героев могут иметь имя "Batman"). Однако по условию задачи требуется вернуть именно имя, поэтому реализован возврат первого подходящего героя с максимальным ростом.

## Структура проекта
* tallest_superhero.py — основной модуль, содержащий функции для загрузки данных, фильтрации и поиска самого высокого героя.

* test_superhero.py — набор тестов для проверки корректности работы всех функций.

* requirements.txt — список зависимостей проекта (для тестов используется pytest, для HTTP-запросов — requests).

* .gitignore — файл для исключения временных и системных файлов из репозитория (виртуальное окружение, кэш Python и pytest).

* README.md — документация проекта.

## Пример использования

```bash

# Самый высокий мужчина с работой
print(get_tallest_hero("Male", True))

# Самая высокая женщина без работы
print(get_tallest_hero("Female", False))
```
